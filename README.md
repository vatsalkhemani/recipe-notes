# 🍲 Recipe Notes

> *Stop losing recipes to WhatsApp chats. Record a voice note, get a clean recipe — forever.*

---

## The Problem

You learn a recipe from someone — your mom, a friend, a relative visiting for the weekend. You either jot it down in a notes app (fragmented), send a voice message to yourself on WhatsApp (buried in chats), or just try to remember it (lost forever).

There's no good way to capture *how people actually share recipes* — verbally, informally, in Hindi and English mixed together — and turn that into something you can actually use in the kitchen.

**Recipe Notes** solves this. You open the app, hit record, let the person explain the dish the way they naturally would, and AI does the rest: transcribes, structures it into ingredients and steps, picks an emoji, suggests tags. The original voice note is preserved alongside the clean recipe so you never lose the nuance.

---

## Who This Is For

Someone who cooks or wants to cook, learns recipes from people (family, friends), and is currently tracking them in WhatsApp, scattered notes, or their head. Primarily designed for mobile (iPhone) but works on desktop too.

---

## Features

### Core recipe experience
- **Card grid** — dishes shown as cards with emoji, name, "learnt from", total cook time, and tags at a glance
- **Voice note playback** — the original recording is preserved and playable inside every recipe
- **Structured recipe view** — ingredients (bulleted), numbered steps with per-step duration, total time, servings, cover photo
- **Full edit mode** — every single field is editable after the fact
- **Cook Mode** — full-screen, distraction-free step-by-step view with a per-step countdown timer and screen wake-lock so your phone doesn't lock mid-cooking

### Adding recipes
- **Voice → AI flow** — record a note → Groq Whisper transcribes it → an open-source LLM structures it into a proper recipe → you review and save (the whole flow takes ~10 seconds)
- **Manual add** — type a recipe by hand if you don't have a recording
- Both flows lead to the same review editor so you can tweak before saving

### Organisation
- **Search** — searches across title, person who taught you, and ingredients
- **Tag filters** — one-tap filter chips (e.g. "breakfast", "vegetarian", "quick") auto-generated by AI and editable
- **Learnt from** — every recipe records who taught you, shown on the card

### Technical
- **Google sign-in** — one tap, your recipes are private to your account
- **Cross-device sync** — add on your phone, view on your laptop (Firebase)
- **Offline support** — already-loaded recipes are readable without internet
- **Installable PWA** — add to iPhone home screen via Safari → Share → Add to Home Screen, works like a native app

## User Flows

### Flow 1: Adding a recipe from a voice note (the main one)
1. Open app → tap **+** (bottom right)
2. Tap **"Record a voice note"**
3. Tap the mic → start recording (the person explains the dish)
4. Tap stop → preview the audio, re-record if needed
5. Tap **"Make recipe"** → ~10 seconds of AI processing
6. The editor opens pre-filled: title, emoji, ingredients, steps with timings, tags
7. Tweak anything → **Save recipe**
8. The card appears in your grid; the voice note is embedded and playable

### Flow 2: Adding a recipe manually
1. Tap **+** → **"Type it manually"**
2. Fill in the editor (dish name, emoji picker, learnt from, servings, tags, ingredients, steps with durations, cover photo)
3. Save → appears in grid

### Flow 3: Cooking from a recipe
1. Tap a card → recipe detail opens
2. Tap **"Start cooking"**
3. Screen 1: ingredients list — get everything ready
4. Tap Next → each step is full-screen, large text
5. If the step has a duration, tap **"Start timer"** for a countdown
6. Tap **Done** when finished

### Flow 4: Finding a recipe
- Type in the search bar (matches title, person who taught you, or any ingredient)
- Tap a tag chip to filter to that category
- Combine both — e.g. "search for 'dal'" + filter by "vegetarian"

### Flow 5: Editing a recipe
1. Tap a card → detail view → tap the **pencil icon**
2. Edit any field inline
3. **Save changes**

## Setup (get it running in ~10 minutes)

### 1. Clone and install
```bash
git clone <your-repo>
cd recipe-notes
npm install
cp .env.local.example .env.local
```

### 2. Firebase — auth + database (5 minutes, free)
1. Go to [console.firebase.google.com](https://console.firebase.google.com) → New project
2. **Build → Authentication** → Get started → Enable **Google** sign-in
3. **Build → Firestore Database** → Create database → Production mode
4. **Project Settings (gear icon)** → Your apps → Add web app → copy the config
5. Paste the 6 values into `.env.local` under the `NEXT_PUBLIC_FIREBASE_*` keys
6. **Firestore → Rules tab** → paste the contents of [`firestore.rules`](firestore.rules) → **Publish**
   (this enforces "a user can only read/write their own recipes"). Production mode
   blocks all access by default, so this step is required or saving will fail.

> We use Firebase only for **Auth + Firestore** (recipe text). File storage is handled
> by Cloudinary below, because Firebase Storage requires a billing card and Cloudinary's
> free tier does not.

### 3. Cloudinary — file storage for audio + photos (3 minutes, free, no card)
1. Sign up at [cloudinary.com/users/register_free](https://cloudinary.com/users/register_free)
2. Copy your **Cloud name** from the dashboard → `CLOUDINARY_CLOUD_NAME`
3. Go to [Settings → Upload → Upload presets](https://console.cloudinary.com/settings/upload/presets) → **Add upload preset** → set **Signing mode: Unsigned** → Save
4. Copy the preset name → `CLOUDINARY_UPLOAD_PRESET`

### 4. Groq API key (2 minutes, free)
1. Go to [console.groq.com/keys](https://console.groq.com/keys) → Create API key
2. Paste it into `.env.local` as `GROQ_API_KEY`
3. That's it — Groq handles both transcription and recipe structuring. **Free tier is plenty.**

> **Note:** A Claude.ai or Claude Code subscription is not an API key. The Anthropic API is a separate pay-per-token product. This app uses Groq instead, which is free for personal use.

### 5. Run
```bash
npm run dev        # opens on http://localhost:3000
npm run test       # unit tests (8 tests)
npm run build      # production build
```

### Install on your iPhone
1. Open `http://localhost:3000` (or your deployed URL) in **Safari**
2. Tap the **Share** icon → **Add to Home Screen**
3. Done — it behaves like a native app, with its own icon

## Try it without setting up any backend

Add `NEXT_PUBLIC_E2E_MODE=true` to `.env.local`. This logs you in as a fake user and stores everything in your browser's localStorage (audio files become embedded data URLs). You won't need Firebase, Cloudinary, or Groq at all — but the voice AI flow will show an error; just use "Type it manually" instead.

This is useful for trying out the UI before committing to the setup.

## Project structure

```
src/
  app/
    api/
      transcribe/   ← Groq Whisper endpoint
      structure/    ← Groq LLM structuring endpoint
      upload/       ← Cloudinary upload proxy (audio + photos)
    layout.tsx      ← root layout, PWA metadata
    page.tsx        ← auth gate
    globals.css     ← warm colour palette + Tailwind
  components/
    add-recipe-flow.tsx   ← record → AI → editor flow
    app-shell.tsx         ← main layout with search + grid
    cook-mode.tsx         ← full-screen cook flow with timers
    recipe-card.tsx       ← grid card
    recipe-detail.tsx     ← detail sheet (view + edit)
    recipe-editor.tsx     ← full editor form
    recipe-grid.tsx       ← responsive card grid
    search-bar.tsx        ← search input + tag chips
    modal-sheet.tsx       ← reusable bottom sheet/modal
    auth-provider.tsx     ← Google auth context
    sign-in-screen.tsx    ← sign-in page
    sw-register.tsx       ← service worker (PWA)
  lib/
    firebase.ts     ← lazy Firebase init (Auth + Firestore)
    recipes.ts      ← Firestore CRUD + Cloudinary upload calls
    types.ts        ← Recipe, Step interfaces + helpers
    ai-prompt.ts    ← LLM system prompt + JSON parser
    use-recorder.ts ← MediaRecorder hook
    e2e-mode.ts     ← localStorage fallback flag
firestore.rules     ← "only you can read your own recipes"
```
